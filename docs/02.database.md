Sarvagya Database Architecture

Purpose

This document defines how Sarvagya interacts with its database.

It serves as the source of truth for database architecture decisions.

---

Database Technology

Primary Database:

PostgreSQL

ORM:

SQLAlchemy 2.0

Migration Tool:

Alembic

Driver:

psycopg

---

Database Philosophy

- PostgreSQL is the single source of truth.
- SQLAlchemy models represent database entities.
- Alembic manages schema evolution.
- Migrations must be version controlled.
- Models should reflect business domains.

---

Database Layer Structure

app/db/

base.py

- Defines the SQLAlchemy Base class.

session.py

- Creates the database engine.
- Configures session factories.

dependencies.py

- Provides FastAPI database dependencies.

Migration Strategy

- All schema changes must go through Alembic.
- Manual database changes are prohibited.
- Migrations must be reviewed before deployment.

---

Guiding Principle

The database should mirror Sarvagya's business domains rather than technical layers.
